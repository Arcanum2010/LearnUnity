This is the GitHub page for the book Learn Unity 4 for iOS Game Development and the accompanying Learn Unity bowling game app.

The GitHub projects, built up cumulatively through each chapter to create a bowling game (similar in style to HyperBowl, do not include assets from the Unity Asset Store and Standard Assets that are used in the book, but you can use them a starting point or to save time in constructing the book examples (but taking shortcuts isn't really recommended). The projects are covered under the MIT open source license.

Summary

The book and app (see below) demonstrate Unity graphics, audio and physics, scripting in JavaScript (and a little bit of C#), touchscreen and accelerometer input, iAd banner and interstitial ads, Game Center leaderboard and achievements, complete rules for a bowling game, and utilizes free Asset Store packages.

The book also covers debugging, profiling and optimization, testing with the Unity Remote and iOS Simulator, build and App Store submission procedures.

Most of the chapters cumulatively build a 3D bowling app with the same control style as HyperBowl, but the Angry Bots demo included with Unity is used to introduce Unity features and a simple spinning cube and dancing skeleton scene is also constructed.

Chapter Descriptions

  1. Downloading and installing Unity, license activation, bug reporting, getting help.
  2. Angry Bots demo, built-in and custom layouts, the Scene, Game, Hierarchy, Inspector and Project Views, how to open a project, open a scene and play a scene.
  3. Creating a new scene with primitive objects, lighting, skybox, camera control, importing Standard Assets, and making prefabs.
  4. Creating scripts, understanding callbacks, editing with MonoDevelop, debugging with MonoDevelop.
  5. Creating a dancing skeleton scene by importing a free skeleton package from the Asset Store, selecting and looping a dance animation, importing and looping music, adding a floor, activating dynamic shadows.
  6. Starting a new bowling game scene: add a bowling ball, add physics materials and rigidbodies, script a physics-based bowing ball controls.
  7. Add bowling pins, a bowling ball game controller, pin collision sounds, ball rolling sound, collision callbacks.
  8. Implement complete bowling game rules as a finite state machine, using coroutines.
  9. Implement a scoreboard and pause menu using UnityGUI.
  10. Build Angry Bots as an iOS app, testing it in the Editor with Unity Remote and on the iOS Simulator in Xcode.
  11. Set up provisioning profiles and test devices on the Provisioning Portal, setting up an app on iTunes Connect, build and run Angry Bots for testing and distribution.
  12. Back to bowling: adding icons and splash screens, specifying screen orientation, scaling UnityGUI for different screen resolutions.
  13. Adding touchscreen controls, detecting shakes with the accelerometer, displaying a video texture with the iOS camera as a webcam.
  14. Adding Game Center leaderboards and achievements.
  15. Adding banner and interstitial (full-screen) ads with iAd.
  16. Running the built-in and Editor-based profiler, optimizing code and assets for speed, size and/or quality.
  17. Misc topics: Android, C#, Editor scripting, script execution order, publishing on the Asset Store.

Corrections and Clarifications

Retail Description

  • Some retail descriptions list Unity's Mecanim animation system. The book does mention but does not use Mecanim. Instead, a chapter is devoted to a dancing skeleton example using Unity's Legacy animation system and a free skeleton package on the Asset Store.
  • Some retail descriptions list Unity's support for gyroscope on iOS devices. The book does mention but does not use Unity's gyroscope API (Input.gyroscope). The book does describe how to access the touchscreen (Input.touches), accelerometer (Input.accelerometer), and webcam (WebCamTexture).
